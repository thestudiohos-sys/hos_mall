## 📅 2026년 3월 9일: 데이터베이스 연동 및 인증 시스템 구현

단순한 UI를 넘어 실제로 데이터를 저장하고 회원을 관리하는 **Backend 연동** 단계입니다.

### 🤝 1. 협업 워크플로우 (Workflow)
우리는 디자인과 개발이 단절되지 않도록 GitHub를 적극 활용합니다.
1.  **디자인 완료**: 이전 단계의 최신 코드를 GitHub에 저장
2.  **배턴 터치**: 안티그래비티(AI)가 GitHub에서 최신 코드를 Pull 하여 작업 시작
3.  **기능 확장**: 기존 디자인 무드를 해치지 않으면서 기능(DB)을 덧붙임

### 🚀 2. Firebase & Auth 도입
웹사이트에 '생명력' 과 '기억력'을 부여했습니다.
*   **Firebase 연동**: 구글 Firebase를 사용하여 DB(Firestore)와 인증(Auth) 시스템 구축
*   **상태 관리**: `AuthContext`를 통해 앱 전체에서 로그인 상태를 실시간 공유
*   **기능 연결**: 
    *   `/signup`: 실제 사용자 계정 생성 및 Firestore `users` 컬렉션 정보 저장
    *   `/login`: 기존 계정 인증 및 세션 유지
    *   `Navbar`: 로그인 상태에 따라 아이콘(로그인/로그아웃) 자동 전환

### 🛠️ 3. 트러블슈팅 (Troubleshooting) - 우리가 해결한 문제들

#### 이슈 #1: "열쇠(API Key)가 없어요!" (`invalid-api-key`)
*   **현상**: 서버 실행 시 빨간색 Firebase 에러 발생
*   **원인**: 보안상 AI가 알 수 없는 실제 API 키가 `.env` 파일에 없기 때문
*   **해결**: 
    1.  Firebase 콘솔에서 키 정보 획득
    2.  `.env` 파일에 `NEXT_PUBLIC_FIREBASE_...` 항목으로 키 입력
    3.  이것은 금고를 설치하고 주인만 아는 열쇠를 꽂는 것과 같은 필수 과정임

#### 이슈 #2: "통로가 이미 사용 중입니다" (`EADDRINUSE: :::9002`)
*   **현상**: `npm run dev` 실행 실패
*   **원인**: 이전 프로세스가 `9002`번 포트를 점유하고 종료되지 않음
*   **해결**: 
    1.  `npx kill-port 9002` 명령어로 해당 포트 강제 해제
    2.  통로를 깨끗이 비운 후 다시 실행하여 해결

### ✅ 4. 시스템 확인 가이드 (Testing)
기능이 정상인지 확인하는 완벽한 시나리오입니다.
1.  **회원가입**: `/signup`에서 가입 후 메인 페이지 상단에 내 이름 툴팁 확인
2.  **로그아웃**: 상단 `LogOut` 아이콘 클릭 후 UI가 초기화되는지 확인
3.  **로그인**: 가입한 계정으로 다시 로그인하여 상태 복구 확인
4.  **DB 확인**: Firebase Console의 Firestore 메뉴에서 데이터가 실시간으로 쌓이는지 확인
